## 🎯 Project Overview

This project was built as part of the **Infrix Remote Internship (Week-2 Task)**:

> *"Scrape data from any website, build an AI-powered chatbot using the collected data, and share your GitHub repository."*

The chatbot scrapes real technology articles from [BBC Technology](https://www.bbc.com/technology) and uses them as a knowledge base. When you ask a question, it retrieves the most relevant articles and feeds them to Groq, which generates a grounded, factual answer.

## 🏗️ Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    RAG Pipeline                             │
│                                                             │
│  BBC Technology                                             │
│       │                                                     │
│       ▼                                                     │
│  scraper.py  ──────────────────►  data/articles.json        │
│  (BeautifulSoup)                  (10–50 articles)          │
│                                         │                   │
│                                         ▼                   │
│  User Query ──► retrieve()  ──► Top-K Articles              │
│                 (keyword                │                   │
│                  scoring)              ▼                    │
│                              build_prompt()                 │
│                                         │                   │
│                                         ▼                   │
│                              Groq llama-3.1-8b-instant        │
│                                         │                   │
│                                         ▼                   │
│                                Grounded Answer              │
└─────────────────────────────────────────────────────────────┘
```

**Why RAG?** Instead of fine-tuning a model (expensive, slow), RAG injects retrieved documents into the prompt at query time. This is the same pattern used in ChatGPT plugins, enterprise AI assistants, and production search systems.

## 🚀 Getting Started

### Prerequisites
- Python 3.13+ recommended
- A Groq API key from [console.groq.com](https://console.groq.com)

The dependency set in this repo was validated on Python 3.13 and 3.14 in this workspace.

### 1. Clone the repository

```bash
git clone https://github.com/hamidnaseem3549/infrix-week-2-tech-news-chatbot.git
cd infrix-week-2-tech-news-chatbot
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Set up your API key

```bash
# Copy the example env file
cp .env.example .env

# Edit .env and paste your Groq API key
# GROQ_API_KEY=gsk_...
```

> **Get a Groq key:** Go to [console.groq.com](https://console.groq.com) → Sign in → create an API key.

### 4. Scrape the data

```bash
python scraper.py
```

This crawls BBC Technology and saves the latest articles to `data/articles.json`. Takes about 30–60 seconds (respectful crawl delay included).

```bash
# Optional: scrape more articles
python scraper.py --max 80
```

### 5. Launch the chatbot

```bash
python -m streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

If your shell already exposes the Streamlit launcher, `streamlit run app.py` also works.

## 🔍 How the Retrieval Works

The chatbot uses **keyword-based scoring** — no embeddings or vector database needed:

```python
score = (
    2 × (query words matching article title) +
    1 × (query words matching article body)
)
```

Top-K scoring articles are passed as context to Groq. Title matches are weighted double because titles are the most signal-dense field. If nothing scores, the most recent articles are used as fallback.

This is intentionally simple and fast — adding semantic search (sentence-transformers + FAISS) would be a natural next improvement.
